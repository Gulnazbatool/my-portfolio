\# My Student Portfolio



A simple project to learn Git and GitHub.



\---



\## ✅ Activity Summary



This project was created to practice essential Git and GitHub operations including version control, branching, merging, conflict resolution, and collaboration.



\---



\## 🔹 Step 1: Setup and First Commit

Initialized a local Git repository, created a README.md file, and performed the first commit.



\---



\## 🔹 Step 2: Connect to GitHub

Created a GitHub repository and linked it with the local repository. Successfully pushed the initial commit to GitHub.



\---



\## 🔹 Step 3: Branching Basics

Created a feature branch (feature-about-me), added about.md file, committed changes, merged into main branch, and deleted the branch.



\---



\## 🔹 Step 4: Multiple Branches and Pull Requests

Created another branch (feature-contact), added contact.md, pushed it to GitHub, created a Pull Request, and merged it into main.



\---



\## 🔹 Step 5: Conflict Resolution Practice

Simulated a merge conflict by editing the same file in different branches. Resolved the conflict manually and completed the merge.



\---



\## 🔹 Step 6: Essential Git Commands

Practiced important Git commands:

\- Undo changes using `git checkout -- file`

\- Viewed commit history using `git log --oneline --graph`

\- Inspected commits using `git show`

\- Used `git stash` to save and restore unfinished work



\---



\## 🔹 Step 7: GitHub Collaboration Features

Created a GitHub Issue for adding an education section. Worked on a new branch (feature-education), added education.md, created a Pull Request, linked it with the Issue, and merged it. The Issue was automatically closed.



\---



\## 🎯 Conclusion

This activity helped in understanding Git workflows, collaboration using GitHub, and handling real-world version control scenarios effectively.
